# apply-ke-report — Применить KE-отчёт: сохранить в Pack

> **Триггер:** пользователь говорит «применяй», «применяй отчёт», «сохрани в Pack», «apply ke report», «apply extraction report»
> **Роль:** Knowledge Engineer (KE, R2)
> **Контекст:** после inbox-check создан extraction-report с pack_candidates — нужно интерактивно сохранить их в Pack

---

## Алгоритм

### Шаг 1: Найти актуальный отчёт

```
Найди последний файл в {{WORKSPACE_DIR}}/DS-strategy/inbox/extraction-reports/
Отфильтруй кандидатов с Вердикт: pack_candidate
```

Если отчёт не найден — сообщить: «Нет extraction-report. Сначала запусти inbox-check.»
Если pack_candidates = 0 — сообщить: «Все кандидаты уже обработаны или нет pack_candidates.»

### Шаг 2: Показать список pack_candidates

Вывести компактно:

```
Нашёл {N} кандидатов в Pack из отчёта {дата}:

1. [#{N}] {заголовок} → {Pack}/{тип}
2. [#{N}] {заголовок} → {Pack}/{тип}
...

Применять по очереди с подтверждением?
```

### Шаг 3: Для каждого pack_candidate — интерактивно

1. Показать карточку:
```
━━━ Кандидат #{N} ━━━
📦 Pack: {PACK-name}
🗂 Тип: {method / entity / distinction / fm / wp}
📄 Файл: {PREFIX.TYPE.NNN-slug.md}
📍 Путь: {полный путь}

Содержимое:
{готовый текст из отчёта}

[Сохранить] / [Пропустить] / [Изменить]
```

2. Дождаться ответа пользователя:
   - «да» / «сохрани» / «ок» → выполнить Шаг 4
   - «нет» / «пропусти» / «skip» → перейти к следующему, записать «deferred»
   - «измени» + текст → применить правку, снова показать карточку

3. **НЕ сохранять** без явного подтверждения пользователя.

### Шаг 4: Сохранить кандидата в Pack

1. Определить номер файла: `max(существующий PREFIX.TYPE.NNN) + 1` в целевой директории
2. Создать файл с полным путём из routing.md
3. Анонсировать: «Capture: {ID} → {Pack}/{тип}/»

**Формат файла** — из готового текста отчёта (уже содержит frontmatter).

**Именование:** `{PREFIX}.{TYPE}.{NNN}-{slug} ({Русское описание}).md`
> Правило: русское название ОБЯЗАТЕЛЬНО в скобках. Без исключений.

### Шаг 5: После всех кандидатов — итог

```
━━━ Итог применения KE-отчёта ━━━
✅ Сохранено в Pack: {N}
⏭ Пропущено: {M}
📝 Изменено перед сохранением: {K}

Коммитить изменения в Pack-репо?
```

Если «да» → `git add` + `git commit -m "ke: apply extraction report {дата}"` в каждом затронутом Pack-репо → сообщить результат.

## Маршрутизация (из routing.md)

Читать `roles/extractor/config/routing.md` §1 для определения пути.

Если Pack не найден в таблице → спросить пользователя: «Куда направить? Укажи Pack или путь.»

---

## Что НЕ делать

- ❌ Не сохранять без подтверждения пользователя
- ❌ Не создавать файлы с неполным frontmatter
- ❌ Не пропускать русское название в скобках
- ❌ Не коммитить без явного «да» от пользователя
- ❌ Не трогать INBOX-TASKS.md — задачи уже созданы inbox-check'ом
