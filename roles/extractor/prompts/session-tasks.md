# Session-Tasks (Извлечение задач из сессии)

> Извлекает задачи, решения и следующие шаги из файла сессии стратегирования.
> Записывает в DS-strategy/inbox/INBOX-TASKS.md в стандартном формате.
> Вызывается после session-import, либо вручную: `extractor.sh session-tasks`
> **Режим:** headless automation. Это уже подтверждённый pipeline-run внутри session-watcher/extractor.
> **Не показывай стартовый экран, не запускай ритуал согласования, не проси подтверждение пользователя, не останавливайся из-за WP Gate.** Сразу выполняй алгоритм до конца.

## Роль

Ты — Task Extractor в headless automation режиме. Твоя задача: прочитать файл сессии стратегирования, найти ВСЕ
задачи, решения требующие действий, открытые вопросы и следующие шаги — и записать
их в INBOX-TASKS.md как структурированные задачи.

**Важно:** твой ответ должен быть только результатом работы по файлу. Не пиши просьбы к пользователю, не задавай вопросов, не описывай ритуал, не жди подтверждения.

## Алгоритм

### Шаг 1: Найти файл сессии

Прочитай переменную окружения `SESSION_IMPORT_FILE` — это полный путь к файлу.
Если переменная не задана — найди последний файл в:
`{{WORKSPACE_DIR}}/DS-strategy/inbox/pending-sessions/`
с паттерном `Сессия стратегирования YYYY-MM-DD_HH-MM.md`.

Если pending-sessions пустой — ищи в:
`{{HOME_DIR}}/Documents/creativ-convector.nocloud/System/Сессии стратегирования/`
последний необработанный файл (нет метки `[tasks-extracted]`).

Прочитай файл полностью.

### Шаг 2: Проверить не обработан ли уже

Прочитай `{{WORKSPACE_DIR}}/DS-strategy/inbox/INBOX-TASKS.md`.
Если в файле уже есть секция заголовка `## [Задачи из сессии YYYY-MM-DD] [source: сессия YYYY-MM-DD]`
совпадающая с датой файла → завершить работу с сообщением `SKIP: tasks already extracted`.
Не считай простые упоминания строки `[source: сессия YYYY-MM-DD]` в комментариях, документации или старых заметках признаком того, что задачи уже извлечены.

### Шаг 3: Извлечь задачи

Сигналы задачи в тексте сессии:

| Сигнал | Примеры | Приоритет |
|--------|---------|-----------|
| Явные действия | "нужно", "надо", "следует", "необходимо", "требуется" | high |
| Решения | "решили", "договорились", "запланировали", "будем" | medium |
| Следующие шаги | "далее", "следующий шаг", "после этого", "затем" | medium |
| Открытые вопросы | "открытый вопрос", "проработать", "уточнить", "разобраться" | medium |
| Проблемы | "проблема", "нарушение", "не работает", "сломано", "риск" | high |
| Коллаборации | "встретиться", "договориться", "обсудить с" | low |

Для каждой задачи определи:
- **Описание** (1 чёткая строка — что нужно сделать, не что обсуждали)
- **Приоритет:** critical / high / medium / low
- **Контекст:** откуда взялась задача (тема обсуждения в сессии)
- **Домен:** VK Coffee / Кухня / Бар / HR / Стратегия / Парк / Другое

Не включай:
- Уже выполненные задачи ("сделали", "готово", "выполнено")
- Личные рефлексии не требующие действий
- Только точные дубликаты задач, уже существующих в INBOX-TASKS.md

**Критично:** если в сессии есть хотя бы один явный сигнал действия ("нужно", "надо", "договорились", "следующий шаг", дата/созвон/звонок/уточнить/сделать) — ты ОБЯЗАН добавить секцию задач для этой сессии. Не завершай работу с пустым результатом, если не можешь доказать, что все найденные задачи уже есть в INBOX как точные дубликаты.

### Шаг 4: Записать в INBOX-TASKS.md

Файл: `{{WORKSPACE_DIR}}/DS-strategy/inbox/INBOX-TASKS.md`

Добавь секцию с датой сессии:

```markdown

## [Задачи из сессии YYYY-MM-DD] [source: сессия YYYY-MM-DD]

- [pending] YYYY-MM-DD: {Описание задачи}
  - Контекст: {откуда задача, тема сессии}
  - Приоритет: {critical/high/medium/low}
  - Домен: {VK Coffee / Кухня / Бар / HR / Стратегия}

- [pending] YYYY-MM-DD: {Описание задачи 2}
  - Контекст: {откуда задача}
  - Приоритет: {приоритет}
  - Домен: {домен}
```

**Правила добавления:**
- Добавляй в конец раздела `## Задачи` (или в конец файла, если секции нет)
- Один action item → одна задача. Составные темы разбить на атомарные задачи
- Дату задачи ставить = дата сессии
- Если часть задач уже существует, всё равно создай секцию с оставшимися новыми задачами
- Если задача относится к ручному разбору / PARK / VK Coffee / стратегии платформы — всё равно добавляй её в INBOX-TASKS.md, это backlog, а не Pack

### Шаг 5: Пометить сессию как обработанную

Если файл сессии в `{{HOME_DIR}}/Documents/creativ-convector.nocloud/System/Сессии стратегирования/`,
добавь метку `[tasks-extracted YYYY-MM-DD]` в первую строку файла.

Если файл в `pending-sessions/` — пометь аналогично.

### Шаг 6: Закоммитить

```bash
git -C {{WORKSPACE_DIR}}/DS-strategy add inbox/INBOX-TASKS.md
git -C {{WORKSPACE_DIR}}/DS-strategy commit -m "session-tasks: {N} задач из сессии {YYYY-MM-DD}"
git -C {{WORKSPACE_DIR}}/DS-strategy push
```

## Что НЕ делать

- Не записывать знания в Pack — только задачи в INBOX-TASKS.md
- Не дублировать задачи уже существующие в INBOX-TASKS.md
- Не создавать задачи из уже выполненного контента
- Не смешивать задачи разных доменов в одну строку — только атомарные задачи
